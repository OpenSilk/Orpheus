Welcome to Orpheus 3

A lot has changed since Orpheus 2. In fact the entire app has been rewritten from scratch. Here are the highlights.

Index

Library starts empty

The first thing you will notice is that you have no music! Orpheus now manages its own index of music using an exclusive policy. So the My Library screen will not have any music in it to start with. You will need to browse through the available Libraries and add the folders where your music is stored. By clicking the action button with the + sign. After indexing the folder it can be removed from My Library by browsing back to the folder and clicking the same action button. (which will now have a - sign)

Why do in this way?

The number one request we have received is folder black-listing. This is now implicitly supported. Don't want a folder in My Library? Just don't index it.

Before only music stored on the sdcard would show in My Library. Now you can index any folder (from any Library plugin you have installed) and have it show in My Library. This is a major new feature in Orpheus 3.

How indexing works

When you index a folder, Orpheus will recursively walk the hierarchy and read the ID3 tags for every audio file it finds. If your music does not have proper ID3 tags indexing will produce poor results. Indexing will not pull cover art from the ID3 tags; however, images in the same directory as the file named cover, artwork, or front will be used as cover art for that album. Also be mindful of indexing over metered connection (3G/LTE). Indexing will download somewhere between 10-50% the total size of the folder. (Indexing A 100GB folder could download 50GB of data!).

Manual re-indexing

Orpheus cannot automatically detect changes in your folders. You must manually rescan them by going to the Folders tab in My Library, long clicking the folder and pressing Rescan in the action menu. This will sync the My Library with any additions and deletions. This is normally very fast. (Auto rescan is on the todo list).

UI changes

The ui has not changed very much since Orpheus 2 but there are some improvements.

Playback controls

The playback controls are now in a right sliding drawer. To access them swipe from the right edge of the screen, or long press on the play button in the footer.

Footer

You can now swipe left or right on the footer to move through the queue.

Now Playing

The Queue and Now Playing screens have been merged into a single screen. Click anywhere on the footer to open it. The queue is a sliding drawer accessible by swiping from the left side of the screen or clicking the navigation button in the action bar.

Now playing doubles as a car mode screen as well, you tell it to keep the screen on by selecting Keep screen on in the action bar menu.

The visualizer (when enabled via the action bar menu), will now only show while music is playing and go back to showing artwork when paused. Note: If you are on Android M (6.0) and revoke the Record Audio permission, the visualizer will not work.

Overflow buttons

The overflow buttons on the list items are gone in favor of an action mode. To enable long click on the list item. This allows performing operations on multiple items at once.

Playlists

Orpheus now manages its own playlists of music in the index. Playlists from Orpheus 2 or other music players will not show up in the playlists screen. This was the only way to support playlists including music from multiple libraries. Which was not supported in Orpheus 2. You can still view and play read only copies of the old playlists from the Android playlists item in the folders library. (Better playlist integration is on the todo list)

Playback

Shuffle

The ordinary shuffle mode is gone. Instead what we do is shuffle the queue then play it in order. This means once a queue is shuffled it cannot be 'unshuffled'.

In addition shuffle is now sticky. After enabling shuffle mode, any new music you enqueue will be added, then the queue will be reshuffled. An important exception to this rule is the play next option, which will insert the items after the current position without shuffling them.

Chromecast

The Cast button in the action bar is gone. To activate casting select Set renderer in the navigation drawer, then select Chromecast. This change is to allow additional renderers in the future. (such as UPNP, but no other renderers exist yet)

Casting is only supported when connected to the same network as the chromecast. This is required because music stored on sdcard is streamed from the phone itself, and cloud sources (like Google Drive) are proxied through the phone. (Since the chromecast is not authorized to view the files). Remote media that does not require authentication (like UPNP) will stream from the source directly. Artwork is always served by the phone.

Last.FM

Artist and album descriptions are now pulled and displayed in the details screens.

External scrobblers are disabled by default. To enable the use of external scrobblers go to Settings -> Audio -> Broadcast meta. (This is what the BROADCAST_STICKY permission is used for). Internal scrobbling and loved tracks are on the todo list.

Translations

Due to the drastic changes made in Orpheus 3 all translations have been removed. If you would like to translate Orpheus into your language of choice please email us using the Send Feedback option in settings.

Closing

Thanks for reading. We hope you enjoy the new Orpheus. And sorry for the no frills intro.
